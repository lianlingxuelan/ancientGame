# Project To-Do

## 提交规则

- Git 提交不频繁做，默认每天最多提交一次。
- 默认提交时间点：每天晚上 10 点左右。
- 如果当天没有新增有效内容，可以不提交。
- 如果第二天继续做了新内容，可以把前一天未提交的有效改动一起提交。
- 提交前必须先检查变更范围，避免把 `all_aseet/` 这类原始素材目录误提交。
- 资源策略：真正进入 Unity 工程并确认会使用的资源，才允许放进 Git 跟踪范围。

## “继续任务”的默认含义

当用户说“继续任务”但没有指定细节时，默认按下面优先级推进：

1. 优先修复当前已知阻塞问题。
2. 其次补齐核心玩法闭环。
3. 再做 UI 表现和美术替换。
4. 最后处理整理、文档、提交等收尾工作。

如果某个动作会扩大范围，例如批量移动资源、提交 Git、删除文件、改后端结构，需要明确说明后再执行。
