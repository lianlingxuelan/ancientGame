# 《神识梦域》项目统一设计基准 v0.1

> 本文档用于统一当前项目方向。后续 UI、玩法、剧情、代码结构、资源取舍，默认按本文档推进；如果有新想法，先更新本文档，再进入开发。

## 1. 项目一句话定位

《神识梦域》是一款以“历史人物陪伴养成 + 强数值成长 + 梦域觉醒”为核心的横屏古风角色养成手游。

玩家作为神魂残缺的上古修行者，通过陪伴历代人物、推进主线与支线、进入人物梦境，逐步回收自身散落在万千时空中的神识碎片。

## 2. 核心爽点排序

当前项目爽点优先级：

1. 角色养成
2. CG / 外观 / 图鉴收集
3. 现实与梦域双主题带来的情绪体验
4. 回合制 PVE 战斗与数值成长
5. 剧情阅读与世界观展开

说明：剧情不是被放弃，而是作为“角色养成、梦域觉醒、情绪爆发”的支撑。主线剧情要精炼，长剧情放入传记、支线、梦域回看。

## 3. 玩家日常核心行为

玩家每天打开游戏，优先做三件事：

1. 养成角色
2. 进入梦域 / 推进梦域内容
3. 领取资源

因此首页和功能入口要服务这三件事，不要一开始堆太多商业化入口。

## 4. 世界观统一设定

### 4.1 主角设定

主角本是上古修行者，因上古大战导致肉身崩毁、神魂碎裂。

散落的神识碎片依附于不同历史人物、传说人物、凡尘众生与梦域生命之中。玩家通过经历他们的人生、遗憾与觉醒，逐步回收神识。

### 4.2 三层世界

1. 现世凡尘
   - 真实历史人物、不同朝代、人文古风。
   - 风格偏清雅、温润、诗意。

2. 众生梦域
   - 人物内心的梦境世界。
   - 承载遗憾、执念、命运分歧与觉醒。
   - 风格偏紫蓝、星尘、蝴蝶、柔光、幻境。

3. 主角自筑梦域
   - 后期解锁。
   - 可容纳原创玄幻角色、妖灵、九天玄女、幻生阵营等内容。
   - 用于扩展非历史人物，不破坏历史人物主线。

## 5. 现实与梦域主题规则

### 5.1 两套主题都保留

现实主题和梦域主题都要做，不做单一选择。

- 现实主题：暖色、米白、浅青、淡金、竹绿、柔粉。
- 梦域主题：淡紫、雾蓝、月白、鎏金、星尘、蝶影。

### 5.2 技术实现原则

第一版采用“同布局换皮肤”的轻量主题方案。

也就是说：

- 页面结构尽量共用；
- 切换主题时更换背景图、边框、按钮图、字体颜色、局部特效；
- 暂时不做两套完全不同的页面布局。

这样后续维护压力小，也方便快速做 Demo。

### 5.3 梦域开启节奏

梦域不是一开始完整开放。

建议节奏：

- 第一章：出现神识波动、梦域链接、觉醒伏笔；
- 中期章节：逐步出现梦域裂隙、角色共鸣；
- 第十章 / 第一卷末章：正式开启梦域系统；
- 后续：梦域伴随支线、角色觉醒、镜像人格、特殊活动玩法。

## 6. 第一章 Demo 范围

第一章 Demo 只围绕李清照。

目标不是做大量内容，而是做出完整体验闭环：

1. 庭院首页
2. 进入主线
3. 主线关卡页
4. 剧情播放
5. 行迹编队
6. 半自动回合战斗
7. 胜利结算
8. 李清照角色养成页
9. 神识波动 / 梦域预告
10. 数据保存到本地后端

第一章时长建议控制在 15-20 分钟。

## 7. 角色获取与抽卡设定

### 7.1 采用混合方案

玩家习惯上更容易接受“获得完整角色”，所以第一版仍以完整角色为主。

但为了贴合世界观，抽卡 / 招募不直接叫普通抽卡，而是包装为：

- 寻访神识
- 寻访灵韵
- 召回碎片
- 梦域镜像
- 幻身结缘

### 7.2 第一版 Demo 暂不做抽卡

第一版先不做正式抽卡系统。

李清照通过主线剧情解锁，后续角色、镜像、幻身、碎片再进入招募系统。

## 8. 角色与养成系统

### 8.1 角色类型

角色来源可以包括：

- 历史人物：李清照、辛弃疾、李白等；
- 传说人物：祝英台等；
- 历史关系人物：唐婉等；
- 原创玄幻角色：主角自筑梦域中诞生；
- 高阶助阵角色：九天玄女等。

### 8.2 第一阶段养成内容

第一阶段先做这些：

- 等级
- 星级 / 境界
- 突破
- 技能升级
- 行迹养成
- 羁绊
- 编队气韵加成

### 8.3 后续扩展

后续再做：

- 装备
- 词意套装
- 梦域镜像
- 神识共鸣
- 属性克制
- 阵营流派
- 助阵神祇

### 8.4 属性命名

外显名称偏诗词化、国风化，例如：

- 气韵
- 词意
- 心魄
- 才情
- 神识
- 墨骨

底层代码可以先用常规 RPG 属性实现，例如：

- 生命
- 攻击
- 防御
- 速度
- 暴击
- 能量

显示层再包装成诗词化名称。

## 9. 战斗系统定位

第一版采用“半自动回合 PVE”。

基础规则：

- 6 人编队；
- 自动普攻；
- 玩家可手动释放技能；
- 支持自动战斗；
- 支持倍速；
- 单场战斗 1-2 分钟；
- 战斗胜利后发放奖励并推进关卡进度。

战斗不是第一版最复杂的部分，但必须能支撑角色养成和数值追求。

## 10. 助阵系统设想

后续可加入“非凡间之人”的助阵系统。

例如：九天玄女作为强力 PVE 助阵位。

原则：

- 只能用于 PVE；
- 效果强大，有改变局势的能力；
- 不能频繁触发；
- 需要玩家通过结缘、梦域进度或特殊资源解锁。

第一版只预留概念，不正式实现。

## 11. 主线关卡视觉规则

主线关卡卡片采用“场景卡”为主，不以人物卡为主。

卡片建议展示：

- 关卡名
- 小标题
- 解锁状态
- 推荐等级 / 推荐战力
- 首通奖励 / 掉落图标

人物图主要放在：

- 角色详情页
- 编队页
- 剧情对话页
- 梦域觉醒页
- 招募页

## 12. UI 与排版原则

### 12.1 推荐主原则

采用“背景弱化 + 内容卡片清晰”的布局方式。

原因：

- 背景图很美，但不能抢文字和按钮；
- 玩家要能快速看懂当前页面能做什么；
- 古风 UI 容易华丽过头，所以信息层级必须清楚。

### 12.2 通用组件必须优先做

后续 UI 不再每个页面单独乱拼，而是逐步形成通用组件：

- 通用按钮
- 通用大面板
- 通用小卡片
- 通用左侧导航
- 通用顶部资源栏
- 通用返回按钮
- 现实主题组件
- 梦域主题组件

### 12.3 字号与颜色

遵循 `world_bg.txt` 中的规范：

- 大章节标题：42px / 古棕
- 二级标题：34px / 浅棕
- 主导航文字：28px
- 普通按钮字：28px
- 主确认按钮字：32px
- 卡片说明和数值：22px
- 备注小字：20px

建议优先补：

## 14. 后端与数据策略

第一阶段使用本地后端 + 本地数据库即可。

需要优先保存：

- 玩家基础信息
- 铜钱 / 玉石 / 体力等资源
- 角色列表
- 角色养成数据
- 关卡进度
- 编队数据
- 剧情阅读进度
- 梦域开启状态
- 任务状态

暂不做：

- 真实充值
- 多账号登录
- 排行榜
- 公会服务器
- 聊天服务器
- 联机玩法

## 15. 代码结构原则

Unity 工程代码要分清楚：

- Editor 脚本：一键生成 UI、整理资源、批量工具；
- Runtime 脚本：游戏运行时真正用的逻辑；
- Data 脚本：读取角色、关卡、剧情、配置；
- Network 脚本：调用后端接口；
- UI 脚本：页面跳转、按钮事件、数据显示；
- Theme 脚本：现实 / 梦域主题切换。

后续所有脚本尽量写中文注释，方便初学阶段理解。
