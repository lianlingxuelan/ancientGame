# Unity 后续开发快速上手

这份文档是给你后续继续做项目时看的。  
目标不是讲原理，而是告诉你“下一步该怎么操作”。

---

## 1. 先记住 Unity 里最重要的几个窗口

### Hierarchy

场景里的对象列表。  
你会在这里看到：

- Main Camera
- Canvas
- 背景图
- 按钮
- 角色图

### Scene

编辑场景的地方。  
你在这里拖拽、摆放、缩放 UI 和图片。

### Game

最终玩家看到的画面预览。  
调 UI 的时候要经常切到这里看效果。

### Project

项目资源管理器。  
图片、脚本、场景、预制体都放这里。

### Inspector

选中对象后，在这里改属性。  
比如：

- 图片大小
- 位置
- 颜色
- 脚本参数

---

## 2. 后续开发的正确顺序

你这个项目后面建议按这个顺序来：

1. 先做场景和 UI 骨架
2. 再做页面切换
3. 再做资源加载
4. 再做角色展示
5. 再做战斗流程
6. 再做养成系统
7. 最后补剧情、活动、抽卡

不要一开始就想把所有系统一次做完。  
先把第一版 Demo 跑通。

---

## 3. 每次做新内容时，先放到哪里

统一放到这些目录：

- `Assets/_Project/Art/Backgrounds`：背景图
- `Assets/_Project/Art/Portraits`：角色头像、立绘
- `Assets/_Project/Art/UI`：UI 图片、按钮、图标
- `Assets/_Project/Art/CG`：CG 图
- `Assets/_Project/Scenes`：场景文件
- `Assets/_Project/Prefabs`：可复用预制体
- `Assets/_Project/Scripts`：代码

以后你只要记住：

- 图放 Art
- 场景放 Scenes
- 代码放 Scripts
- 可复用物体放 Prefabs

---

## 4. 新增图片时怎么做

### 4.1 导入图片

把图片直接拖进 Unity 左下角的 `Project` 窗口，放进对应文件夹。

### 4.2 设置图片类型

选中图片后，在 `Inspector` 里检查：

- 如果是 UI 背景图，通常用默认设置就行
- 如果后面要做头像或切图，再改成 Sprite 模式

### 4.3 拖到场景里

如果是背景图：

- 先建 `Canvas`
- 再建 `Image`
- 把图片拖到 `Source Image`

如果是角色图：

- 后面放到角色展示层

---

## 5. 如何做一个新的 UI 页面

你现在建议统一用“页面”思路。

### 5.1 页面命名

统一格式：

- `PageHomeView`
- `PageCharacterView`
- `PageBattleView`
- `PageStoryView`
- `PageActivityView`

### 5.2 页面对象结构

每个页面建议是：

- 一个根节点
- 一个背景
- 若干按钮
- 若干文本

### 5.3 创建新页面的步骤

1. 先在 `Canvas` 下建一个空物体
2. 改名为 `PageXxx`
3. 在里面放图片、按钮、文字
4. 最后再挂脚本

---

## 6. 如何改页面脚本

你现在已经有基础脚本了。

### 6.1 脚本放哪

放到：

- `Assets/_Project/Scripts/UI`
- `Assets/_Project/Scripts/Core`
- `Assets/_Project/Scripts/Data`

### 6.2 什么时候改脚本

你要在这些情况改脚本：

- 要切换页面
- 要点按钮
- 要显示/隐藏 UI
- 要加载角色数据
- 要更新文本
- 要显示战斗信息

### 6.3 改完脚本后

保存后，回 Unity 等它自动编译。  
如果有红字报错，先看 `Console`。

---

## 7. Console 报错怎么看

这是 Unity 最常见的排错地方。

### 常见问题

- 脚本名字和类名不一样
- 引用空了
- 文件路径错误
- 缺少 `using`
- 场景里没挂脚本

### 处理顺序

1. 先看最上面那条红字
2. 双击错误跳到对应代码
3. 先修语法错误
4. 再修引用错误

---

## 8. 场景文件怎么管理

当前推荐：

- `Scene_Boot`：启动场景
- `Scene_Home`：主界面
- `Scene_Battle`：战斗
- `Scene_Login`：登录

### 新建场景的方法

1. 打开 `File > New Scene`
2. 搭好场景内容
3. `File > Save As`
4. 存到 `Assets/_Project/Scenes`
5. 按统一名字保存

---

## 9. 怎么做页面切换

你后面会经常做这件事。

### 思路

一个页面显示，其他页面隐藏。

### 典型流程

- 进入游戏后显示 `Home`
- 点角色按钮，切到 `Character`
- 点战斗按钮，切到 `Battle`
- 点剧情按钮，切到 `Story`

### 开发原则

不要一开始把所有页面都塞到一起。  
先做 2 到 3 个页面，确认流程能跑。

---

## 10. 做主界面时的推荐顺序

你现在最适合这样做：

1. 先放背景图
2. 再放顶栏
3. 再放底部导航
4. 再放中央角色位
5. 再补按钮和文字
6. 最后做动画和交互

不要反过来。  
先把结构做出来，再美化。

---

## 11. 后续最常用的操作

### 11.1 保存场景

`Ctrl + S`

### 11.2 运行预览

点上方的播放按钮 `Play`

### 11.3 停止预览

再点一次 `Play`

### 11.4 复制对象

`Ctrl + D`

### 11.5 撤销

`Ctrl + Z`

---

## 12. 你以后做新功能时的标准流程

每次新增一个功能，都按这个顺序走：

1. 先想清楚这个功能是做什么的
2. 确定它属于哪个页面
3. 决定它放哪个文件夹
4. 先摆 UI
5. 再挂脚本
6. 最后测试

这样不会乱。

---

## 13. 适合你这个项目的开发节奏

你现在是个人开发，建议节奏如下：

- 先做第一章 Demo
- 先完成首页
- 再做角色页
- 再做战斗页
- 再做剧情页
- 再做活动页

不要一口气做完整项目。  
先做“能玩的一小段”。

---

## 14. 一句话记忆版

你只要记住这句：

> 先搭骨架，再放内容；先做能跑的 Demo，再慢慢补美术和系统。
